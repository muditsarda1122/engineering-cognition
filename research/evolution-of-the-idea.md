# Evolution of the Idea

Engineering Cognition did not begin as an attempt to define a new research direction.

It emerged while trying to solve an engineering problem.

## Phase 1 — Engineering Continuity

The initial observation was simple.

Every time an AI coding session ended, the next session began almost from scratch.

The immediate objective became preserving continuity between sessions.

The working hypothesis was:

> Better continuity would improve software development.

---

## Phase 2 — Engineering Memory

This naturally led to the idea of an Engineering Memory System (EMS).

The goal shifted from preserving sessions to preserving engineering knowledge.

The implementation became centered around:

- session summaries
- persistent knowledge
- repository history
- architectural decisions
- debugging discoveries

At this stage, memory appeared to be the missing abstraction.

---

## Phase 3 — Understanding the Wrong Question

While building EMS, an important realization emerged.

Human engineers do not accumulate conversations.

They accumulate understanding.

Memory was only preserving artifacts.

The real objective was preserving the engineering understanding those artifacts represented.

---

## Phase 4 — Engineering Cognition

This shifted the research question entirely.

Instead of asking:

> How should engineering memory be stored?

the project began asking:

> What should actually be stored?

This eventually became the question:

> What is Engineering Cognition?

Today, EMS is viewed as one experimental implementation.

Engineering Cognition is the research problem.
