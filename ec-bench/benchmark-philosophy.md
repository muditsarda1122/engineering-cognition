# Benchmark Philosophy

EC-Bench is not intended to measure code generation.

It is intended to measure accumulated engineering understanding.

Most existing software engineering benchmarks evaluate whether a model can solve a task.

EC-Bench instead evaluates whether solving previous tasks improves performance on future tasks.

The benchmark therefore attempts to simulate software engineering as it actually occurs:

- investigation
- implementation
- debugging
- design
- feature evolution

rather than isolated question answering.
